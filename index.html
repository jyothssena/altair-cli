Altair

The Automagic Documenter

A multi-pass code intelligence pipeline that transforms raw git diffs into structured documentation using Google Gemma 3 via Ollama.

GDG Hackathon 2026 Track 1: Git-to-Doc Gemma 3 (4B) Local-First

The Problem

Developers write terrible commit messages, skip changelogs, and rarely document their code changes. Manual documentation is friction that slows teams down and makes codebases unreadable over time.

The Solution

Altair takes a raw .diff file and runs it through a 5-stage intelligent pipeline powered by Gemma 3, producing structured commit messages, changelogs, code reviews, and PR descriptions — automatically.

Pipeline Architecture

5 stages from raw diff to structured documentation

STAGE 1

Pre-Analysis Engine

Pure Python metadata extraction — languages, complexity, change types. No LLM needed.

STAGE 2

Smart Chunker

Function-boundary splitting. Large diffs split by def/class, preserving context within model limits.

STAGE 3 Multi-Pass LLM Pipeline

Commit

Conventional Commit format

Changelog

Keep a Changelog entry

Review

Risk + issues + security

PR Desc

Title + summary + testing

STAGE 4

Confidence Scoring

Validates output structure. HIGH/MEDIUM/LOW confidence. Auto-retries on failure (up to 3x).

OUTPUT

Structured Documentation

Terminal UI • JSON • Markdown file • Git hook auto-inject

Features

Built for real developer workflows

Multi-Pass Pipeline

4 specialized LLM passes with dedicated prompts for commit, changelog, review, and PR description.

Pre-Analysis Engine

45+ language detection, complexity scoring (1-10), change type inference — all before the LLM call.

Smart Chunking

Function-boundary splitting for large diffs. Hunk-aware truncation within model context limits.

Confidence Scoring

Validates LLM output structure, assigns confidence levels, auto-retries on low scores.

Batch Mode

Process entire branch range into consolidated release notes with per-commit timing.

Git Hook

Auto-generates commit messages on every git commit via prepare-commit-msg hook.

Usage

Simple CLI — one command, full documentation

Full Pipeline 
$ python gemma_ollama.py input.diff

# Single pass only
$ python gemma_ollama.py input.diff --pass commit
$ python gemma_ollama.py input.diff --pass changelog
$ python gemma_ollama.py input.diff --pass review
$ python gemma_ollama.py input.diff --pass pr
Advanced 
# Pipe git diff directly
$ git diff | python gemma_ollama.py -

# JSON output for CI/CD
$ python gemma_ollama.py input.diff --json

# Batch release notes
$ python gemma_ollama.py --batch main..feature

# Install git hook
$ python gemma_ollama.py --install-hook

Design Decisions

Why we built it this way

01

Local-First

Everything runs on-device via Ollama. No cloud APIs, no data leaves the machine. Privacy by default.

02

Pre-Analysis Before LLM

Structured metadata reduces hallucination and gives the model context it can't infer from raw diffs alone.

03

Retry with Validation

Don't trust LLM output blindly. Validate structure, score confidence, and retry on failure — up to 3 times.

04

Model-Agnostic

Works with any Ollama model via --model flag. Defaults to Gemma 3 4B but supports Llama, Qwen, etc.

05

Modular Architecture

12 focused modules in the gitdoc/ package. Each component is independently testable and swappable.
